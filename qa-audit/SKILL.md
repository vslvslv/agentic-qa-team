## Preamble (run first)

### Skill arguments

If the user invoked this skill with `--since=<git-ref>` (a commit SHA, branch name, or
tag), set `_SINCE_REF` to that value below. Otherwise leave it empty. `--since=` enables
**delta mode**: only test files changed since `<git-ref>` are inventoried and scored,
instead of the entire test tree. Useful for per-PR audits — `/qa-audit --since=main`.

```bash
_TMP="${TEMP:-${TMP:-/tmp}}"
_DATE=$(date +%Y-%m-%d)
_BRANCH=$(git branch --show-current 2>/dev/null || echo "unknown")
_SINCE_REF=""           # ← if user passed --since=<ref>, set it here
_DELTA_MODE=0
_SINCE_SHA=""
_CHANGED_TEST_FILES=""
echo "BRANCH: $_BRANCH"

# --- Delta-mode resolution ---
if [ -n "$_SINCE_REF" ]; then
  _SINCE_SHA=$(git rev-parse --short --verify "$_SINCE_REF^{commit}" 2>/dev/null || true)
  if [ -z "$_SINCE_SHA" ]; then
    echo "ERROR: --since=$_SINCE_REF does not resolve to a git commit. Aborting."
    exit 1
  fi
  if ! git merge-base --is-ancestor "$_SINCE_SHA" HEAD 2>/dev/null; then
    echo "ERROR: --since=$_SINCE_REF ($_SINCE_SHA) is not an ancestor of HEAD. Aborting."
    exit 1
  fi
  # Test-file pattern matches qa-team-suggest-rerun + Phase 5 verify loop.
  _CHANGED_TEST_FILES=$(git diff --name-only "$_SINCE_SHA"..HEAD 2>/dev/null \
    | grep -E '\.(test|spec)\.[jt]sx?$|_test\.py$|^test_.*\.py$|Tests?\.cs$|_spec\.rb$|_test\.go$|Test\.java$|Tests\.java$' \
    || true)
  _CHANGED_COUNT=$(printf '%s\n' "$_CHANGED_TEST_FILES" | grep -c . || true)
  _DELTA_MODE=1
  echo "DELTA_MODE: enabled (since $_SINCE_REF → $_SINCE_SHA)"
  echo "DELTA_CHANGED_TEST_FILES: $_CHANGED_COUNT"
  if [ "${_CHANGED_COUNT:-0}" -eq 0 ]; then
    echo "No test files changed since $_SINCE_REF — emitting no-op result and exiting."
    # Skip Phases 1-4 (nothing to score). Still write the JSON/report so consumers
    # see the run. Phase 5 will branch on _DELTA_MODE + _CHANGED_COUNT==0.
  fi
fi

# --- Test file discovery by layer ---
echo "--- TEST INVENTORY ---"
_UNIT=$(find . \( \
  -path "*/unit/*.spec.*" -o -path "*/unit/*.test.*" \
  -o -name "*.unit.spec.*" -o -name "*.unit.test.*" \
  -o -path "*/__tests__/unit*" \
  \) ! -path "*/node_modules/*" 2>/dev/null | wc -l | tr -d ' ')
_INTEG=$(find . \( \
  -path "*/integration/*.spec.*" -o -path "*/integration/*.test.*" \
  -o -name "*.integration.spec.*" -o -name "*.integration.test.*" \
  -o -path "*/__tests__/integration*" \
  \) ! -path "*/node_modules/*" 2>/dev/null | wc -l | tr -d ' ')
_E2E=$(find . \( \
  -path "*/e2e/*.spec.*" -o -path "*/e2e/*.test.*" \
  -o -name "*.e2e.spec.*" -o -name "*.e2e.test.*" \
  -o -path "*/tests/e2e*" \
  \) ! -path "*/node_modules/*" 2>/dev/null | wc -l | tr -d ' ')
_ALL_TESTS=$(find . \( \
  -name "*.spec.ts" -o -name "*.spec.js" \
  -o -name "*.test.ts" -o -name "*.test.js" \
  -o -name "*_test.py" -o -name "test_*.py" \
  -o -name "*Test.java" -o -name "*_spec.rb" \
  \) ! -path "*/node_modules/*" 2>/dev/null | wc -l | tr -d ' ')
echo "UNIT: $_UNIT  INTEGRATION: $_INTEG  E2E: $_E2E  TOTAL: $_ALL_TESTS"

# --- Delta-mode inventory override ---
# In delta mode, the relevant tests are *only* the ones that changed since _SINCE_SHA.
# Override the counts so Phase 4 scores against that subset.
if [ "$_DELTA_MODE" = "1" ]; then
  _ALL_TESTS=$(printf '%s\n' "$_CHANGED_TEST_FILES" | grep -c . || true)
  _UNIT=$(printf '%s\n' "$_CHANGED_TEST_FILES" | grep -cE '/unit/|\.unit\.' || true)
  _INTEG=$(printf '%s\n' "$_CHANGED_TEST_FILES" | grep -cE '/integration/|\.integration\.' || true)
  _E2E=$(printf '%s\n' "$_CHANGED_TEST_FILES" | grep -cE '/e2e/|\.e2e\.' || true)
  echo "DELTA_OVERRIDE: UNIT: $_UNIT  INTEGRATION: $_INTEG  E2E: $_E2E  TOTAL: $_ALL_TESTS"
fi

# --- Framework detection ---
echo "--- FRAMEWORK ---"
grep -ql '"jest"\|"vitest"\|"@testing-library"' package.json 2>/dev/null && echo "FRAMEWORK: jest/vitest (JS/TS)"
ls pytest.ini conftest.py pyproject.toml 2>/dev/null | head -1 | grep -q '.' && echo "FRAMEWORK: pytest (Python)"
ls pom.xml build.gradle 2>/dev/null | head -1 | grep -q '.' && echo "FRAMEWORK: JUnit (Java)"
ls Gemfile 2>/dev/null | grep -q '.' && echo "FRAMEWORK: RSpec (Ruby)"

# --- Coverage configuration ---
echo "--- COVERAGE ---"
grep -ql '"coverage"\|"nyc"\|"c8"' package.json 2>/dev/null && echo "COVERAGE_TOOL: nyc/c8"
ls .nycrc .nycrc.json jest.config.ts jest.config.js vitest.config.ts vitest.config.js 2>/dev/null | head -3
grep -r "coverageThreshold" --include="*.json" --include="*.ts" --include="*.js" \
  ! -path "*/node_modules/*" 2>/dev/null | head -3
grep -ql "pytest-cov\|coverage\[" requirements.txt pyproject.toml 2>/dev/null && echo "COVERAGE_TOOL: pytest-cov"
grep -r "\-\-cov-fail-under" --include="*.ini" --include="*.cfg" --include="*.toml" \
  2>/dev/null | head -3

# --- Flakiness signals ---
echo "--- FLAKINESS SIGNALS ---"
_SLEEP_COUNT=$(grep -r "sleep\|setTimeout\|Thread\.sleep\|time\.sleep" \
  --include="*.spec.*" --include="*.test.*" --include="*_test.*" \
  ! -path "*/node_modules/*" 2>/dev/null | wc -l | tr -d ' ')
_RETRY_COUNT=$(grep -r "retry\|retries\|flaky\|\.skip\|xtest\|xit\b" \
  --include="*.spec.*" --include="*.test.*" \
  ! -path "*/node_modules/*" 2>/dev/null | wc -l | tr -d ' ')
echo "SLEEP_CALLS: $_SLEEP_COUNT  RETRY_MARKS: $_RETRY_COUNT"

# --- BDD signals ---
echo "--- BDD ---"
_FEATURES=$(find . -name "*.feature" ! -path "*/node_modules/*" 2>/dev/null | wc -l | tr -d ' ')
echo "FEATURE_FILES: $_FEATURES"
ls cucumber.js cypress.json jest-cucumber.config.* 2>/dev/null | head -2

# --- CI config ---
echo "--- CI ---"
ls .github/workflows/*.yml .circleci/config.yml .gitlab-ci.yml azure-pipelines.yml \
  Jenkinsfile 2>/dev/null | head -5

echo "--- DONE ---"
```

If `_ALL_TESTS` is 0: warn the user that no test files were found. Recommend starting
with `/qa-methodology-refine test-pyramid` to understand testing strategy. Skip Phases
2–4 and produce a "No tests found" report.

> **Delta mode (`_DELTA_MODE=1`)**: skip the find-based sampling below. Iterate over

Sample test files across all layers. Read up to 3 files from each detected layer:

```bash
# Sample unit tests

# Sample e2e tests
find . \( -path "*/e2e/*.spec.*" -o -name "*.e2e.spec.*" \) \

# Sample unclassified tests (top-level spec files)
find . -maxdepth 4 \( -name "*.spec.ts" -o -name "*.spec.js" -o -name "*.test.ts" \) \
  ! -path "*/node_modules/*" ! -path "*/e2e/*" ! -path "*/unit/*" ! -path "*/integration/*" \
  2>/dev/null | head -5
```

Read the sampled files. For each, note the observed pattern and classify:

**Layer classification heuristics** (when file path isn't conclusive):
- Imports a real HTTP client (`fetch`, `axios`, `requests`, `RestAssured`) without mocking → **integration**
- Imports `playwright`, `cypress`, `selenium`, `detox`, `appium` → **e2e**
- Mocks all I/O dependencies → **unit**
- Mix of real and mocked dependencies → **integration**

Build the inventory table:

```
Layer        | Count | % of total | Sample files
-------------|-------|------------|-------------
Unit         |   N   |   N%       | path/to/file.spec.ts
Integration  |   N   |   N%       | path/to/file.spec.ts
E2E          |   N   |   N%       | path/to/file.spec.ts
Unclassified |   N   |   N%       | path/to/file.spec.ts
```

Ideal pyramid: **unit > 60%**, **integration 20–35%**, **e2e < 20%**.

## Phase 2 — Methodology Checks

Run these static checks and record concrete findings with file locations.

**Check 1: Test naming quality**

```bash
# Find vague test names
grep -rn "it\(\|test\(\|def test_\|@Test" \
  --include="*.spec.*" --include="*.test.*" --include="*_test.*" \
  ! -path "*/node_modules/*" 2>/dev/null | \
  grep -iE "'test[0-9]*'|\"test[0-9]*\"|'it works'|'should work'|'does something'|'test this'" | head -10
```

Good: `"returns 401 when token is expired"` — describes behavior + expected outcome.
Bad: `"test auth"`, `"it works"`, `"test1"`.

**Check 2: Arrange-Act-Assert / Given-When-Then structure**

```bash
grep -rn "// Arrange\|// Act\|// Assert\|// Given\|// When\|// Then\|#arrange\|#act\|#assert" \
  --include="*.spec.*" --include="*.test.*" --include="*_test.*" \
  ! -path "*/node_modules/*" 2>/dev/null | wc -l
```

Low count doesn't mean bad tests — read 3 unstructured test files to see if the
structure is implied but unwritten.

**Check 3: Test isolation — shared mutable state**

```bash
# let/var at describe scope (not reset in beforeEach) — JS/TS
grep -rn "^\s*let \|^\s*var " \
  --include="*.spec.ts" --include="*.spec.js" --include="*.test.ts" \
  ! -path "*/node_modules/*" 2>/dev/null | head -10

# beforeEach cleanup presence
grep -rn "beforeEach\|afterEach\|setUp\|tearDown\|setup_method\|teardown_method" \
  --include="*.spec.*" --include="*.test.*" --include="*_test.*" \
  ! -path "*/node_modules/*" 2>/dev/null | wc -l
```

**Check 4: Sleep / timing dependencies (flakiness risk)**

```bash
grep -rn "sleep\|setTimeout.*[0-9]\{3,\}\|Thread\.sleep\|time\.sleep\|asyncio\.sleep" \
  --include="*.spec.*" --include="*.test.*" --include="*_test.*" \
  ! -path "*/node_modules/*" 2>/dev/null | head -10
```

Each hardcoded sleep in a test is a flakiness risk. Report count + locations.

**Check 5: Test data strategy**

```bash
# Hardcoded literals in test bodies
_HARDCODED=$(grep -r "@example\.com\|password123\|test-user-[0-9]\|admin@\|\"admin\"" \
  --include="*.spec.*" --include="*.test.*" --include="*_test.*" \
  ! -path "*/node_modules/*" 2>/dev/null | wc -l | tr -d ' ')

# Factory / fixture usage
_FACTORY=$(grep -r "factory\|faker\|fixture\|\.build(\|\.create(\|Factory\.\|Faker\." \
  --include="*.spec.*" --include="*.test.*" --include="*_test.*" \
  ! -path "*/node_modules/*" 2>/dev/null | wc -l | tr -d ' ')

echo "HARDCODED: $_HARDCODED  FACTORY: $_FACTORY"
```

**Check 6: Coverage configuration**

Read the coverage config file (jest.config.ts, vitest.config.ts, pytest.ini, etc.).
Check:
- Coverage enabled: yes/no
- Thresholds configured: yes/no (if yes, what values?)
- Branch coverage measured: yes/no

**Check 7: CI test integration**

Read the first CI config file found. Check:
- Tests run on every PR/push: yes/no
- Tests gate merge (required status check): inferred
- Parallel test execution: yes/no
- Separate stages (fast unit first, slow e2e last): yes/no

## Phase 4 — Score

Score 5 dimensions × 20 points each (100 total):

| Dimension | 0 | 10 | 20 | Signal |
|-----------|---|----|----|--------|
| Pyramid Balance | Inverted or all-e2e (> 50% e2e) | Off-balance but improving | Unit > 60%, e2e < 20% | Computed from inventory |
| Test Isolation | Shared state + no cleanup | Some isolation | beforeEach resets + no cross-test deps detectable | Check 3 |
| Test Data | All hardcoded values | Some factories | Factory/fixture ratio > hardcoded ratio | Check 5 |
| Naming Quality | > 30% vague names | Some descriptive | < 5% vague names, all describe behavior | Check 1 |
| CI / Coverage | No CI or no coverage | CI runs tests | CI + coverage thresholds + branch coverage | Checks 6 & 7 |

A score of 50–65 is typical for projects that haven't explicitly focused on test
methodology. Report the score with context, not judgment.

## Phase 5c — Persist to Project History

Copy the JSON sidecar (and the markdown report) into a project-local history directory so
trend analysis and per-commit comparisons survive across runs. Skip silently when the
working directory is not a git repo.

**Important — delta runs are transient.** When `_DELTA_MODE=1`, **skip persistence
entirely**: do not write to `.qa-team/`, do not update `qa-audit-latest.json`. The history
directory holds full-audit snapshots that are comparable across runs; mixing in delta
scores would corrupt the trend rendered by `bin/qa-team-history` and the regression
detection in `qa-team` Phase 5.

```bash
if [ "$_DELTA_MODE" = "1" ]; then
  echo "HISTORY: delta-mode run — skipping persistence (transient by design)"
else
  _REPO_ROOT="$(git rev-parse --show-toplevel 2>/dev/null)"
  if [ -n "$_REPO_ROOT" ]; then
    _HIST_DIR="$_REPO_ROOT/.qa-team"
    mkdir -p "$_HIST_DIR"
    _HIST_KEY="qa-audit-${_COMMIT}-$(date -u +%Y%m%dT%H%M%SZ)"
    cp "$_TMP/qa-audit-score.json"  "$_HIST_DIR/${_HIST_KEY}.json"  2>/dev/null || true
    cp "$_TMP/qa-audit-report.md"   "$_HIST_DIR/${_HIST_KEY}.md"    2>/dev/null || true
    ln -sf "${_HIST_KEY}.json" "$_HIST_DIR/qa-audit-latest.json"    2>/dev/null || true
    echo "HISTORY: persisted to $_HIST_DIR/${_HIST_KEY}.{json,md}"
  fi
fi
```

The first non-delta run on a project creates `.qa-team/`. Add it to `.gitignore` if you do
not want the history committed; or commit it deliberately to track score over time
alongside code.

## Important Rules

- **Read actual test files** — scan content, not just file counts; a directory named `unit/` may contain integration tests
- **Concrete examples** — every recommendation must show actual code from the project (before) and the fix (after)
- **Report even with 0 tests** — produce the report; recommend starting with `/qa-methodology-refine test-pyramid`
- **No judgment on score** — a low score means opportunity, not blame; frame positively
- **Link to guides** — when a methodology guide exists for a finding, reference it explicitly
- **Idempotent** — safe to re-run; always overwrites `$_TMP/qa-audit-report.md` and `$_TMP/qa-audit-score.json`
- **JSON contract is load-bearing** — `qa-audit-score.json` is consumed by `qa-team`'s verify-after-fixes phase, by `bin/qa-team-history`, and by user-defined CI hooks. Do not change field names without bumping `schema_version` and updating consumers
- **Delta mode (`--since=<ref>`) is transient** — delta runs do NOT persist to `.qa-team/` and do NOT update `qa-audit-latest.json`. Their score is over a subset, not the whole suite, and treating them as snapshots would corrupt trend rendering. Only full audits go in the history directory.
